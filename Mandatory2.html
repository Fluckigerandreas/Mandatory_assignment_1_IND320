import requests
import pandas as pd
from datetime import datetime
from dateutil.relativedelta import relativedelta

# --- API Settings ---
BASE_URL = "https://api.elhub.no/energy-data/v0/price-areas"
DATASET = "PRODUCTION_PER_GROUP_MBA_HOUR"
START_DATE = "2021-01-01"
END_DATE = "2021-12-31"

# --- Step 1: Get all available price areas dynamically ---
areas_response = requests.get(BASE_URL)
if areas_response.status_code == 200:
    areas = areas_response.json().get("priceAreas", ["NO1", "NO2", "NO3", "NO4", "NO5"])
else:
    print(" Could not fetch price areas from API. Using defaults (NO1-NO5).")
    areas = ["NO1", "NO2", "NO3", "NO4", "NO5"]

print(f" Price areas to fetch: {areas}")


# --- Step 2: Function to fetch data for one area and time window ---
def fetch_data(area, start_date, end_date):
    api_url = f"{BASE_URL}/{area}"
    params = {
        "dataset": DATASET,
        "startDate": start_date,
        "endDate": end_date
    }
    response = requests.get(api_url, params=params)

    if response.status_code == 200:
        try:
            data = response.json()
            records = data["data"][0]["attributes"]["productionPerGroupMbaHour"]
            if records:
                df = pd.DataFrame(records)
                df["priceArea"] = area
                return df
            else:
                print(f" No data returned for {area}: {start_date}  {end_date}")
                return pd.DataFrame()
        except (KeyError, IndexError):
            print(f" Unexpected JSON structure for {area}: {start_date}  {end_date}")
            return pd.DataFrame()
    else:
        print(f" Failed for {area}: {start_date}  {end_date} ({response.status_code})")
        return pd.DataFrame()


# --- Step 3: Loop over all months and all areas ---
all_data = []
start_dt = datetime.fromisoformat(START_DATE + "T00:00:00+02:00")
end_dt = datetime.fromisoformat(END_DATE + "T23:59:59+02:00")

for area in areas:
    current_start = start_dt
    while current_start < end_dt:
        current_end = current_start + relativedelta(months=1)
        if current_end > end_dt:
            current_end = end_dt

        start_str = current_start.isoformat()
        end_str = current_end.isoformat()

        print(f"Fetching {area}: {start_str}  {end_str}")
        df = fetch_data(area, start_str, end_str)
        if not df.empty:
            all_data.append(df)

        current_start = current_end


# --- Step 4: Combine and save all data ---
if all_data:
    final_df = pd.concat(all_data, ignore_index=True)
    final_df.to_csv("elhub_production_2021_all_areas.csv", index=False)
    print(f" Data for all price areas saved to elhub_production_2021_all_areas.csv")
else:
    print(" No data collected.")

import pandas as pd
import matplotlib.pyplot as plt

# --- Choose your price area ---
chosen_area = "NO1"

# --- Filter for the chosen price area ---
df_area = pdf[pdf["pricearea"] == chosen_area].copy()

# Convert time columns to datetime
df_area["starttime"] = pd.to_datetime(df_area["starttime"])

# -----------------------------
# 1 Pie chart: total production by production group for the year
# -----------------------------
total_by_group = df_area.groupby("productiongroup")["quantitykwh"].sum()

# Define colors (similar to Plotly qualitative Set3)
colors = plt.get_cmap('Set3').colors  # Matplotlib has Set3 colormap

plt.figure(figsize=(7,7))
plt.pie(
    total_by_group,
    labels=total_by_group.index,
    autopct='%1.1f%%',
    colors=colors[:len(total_by_group)],
    startangle=90
)
plt.title(f"Total Production in {chosen_area} (Year)")
plt.axis('equal')  # Equal aspect ratio ensures pie is circular
plt.show()

# -----------------------------
# 2 Line plot: first month of the year
# -----------------------------
df_january = df_area[df_area["starttime"].dt.month == 1]

plt.figure(figsize=(12,6))
for i, group in enumerate(df_january["productiongroup"].unique()):
    group_data = df_january[df_january["productiongroup"] == group]
    plt.plot(
        group_data["starttime"],
        group_data["quantitykwh"],
        label=group,
        marker='o',
        color=colors[i % len(colors)]
    )

plt.title(f"Hourly Production in {chosen_area} (January)")
plt.xlabel("Time")
plt.ylabel("Quantity (kWh)")
plt.grid(True)
plt.legend(title="Production Group")
plt.tight_layout()
plt.show()

Work description and log

For the second compulsory assignment, a significant portion of time was dedicated to resolving dependency issues. After extensive trial and error, we successfully configured the environment using Spark 3.5.0, NumPy 2.3.4, and Python 3.11.9. This setup has proven stable and reliable for the tasks at hand, allowing us to focus more on the actual data processing rather than environment issues. It was a valuable learning experience in handling version conflicts and understanding the subtle compatibility requirements of modern data science libraries.

For data visualization, I opted to use Plotly. The interactive nature of Plotly plots provided a more dynamic exploration of the data, making it easier to gain insights and compare different subsets of the dataset. Hover information, zooming capabilities, and the ability to filter data interactively greatly enhanced the efficiency of the analysis. This approach contrasts with static plots, which can be less flexible when analyzing large and complex datasets. For the notebook, I had to use matplotlib because of the conversion to pdf-file for the hand-in.

Regarding data storage and management, the following code was used to write data to Cassandra:

After developing the logic, I extracted the filtered data from Cassandra to create the pdf DataFrame, which serves as the basis for further analysis. The process of transferring data to MongoDB was completed without any issues. Integrating the results into Streamlit for interactive visualization worked seamlessly, enabling efficient exploration and sharing of insights. To improve performance, I implemented a st.cache function to reduce data loading times. For the line plot, I chose to aggregate production across all price areas selected by the user, making the visualization both cleaner and more informative. I also managed to send the data regarding price area NO1 twice to MongoDB, hence the duplication part of the code in streamlit.

In addition, I encountered some issues related to GitHub password security. Since the GitHub account was used to store scripts and datasets, ensuring strong passwords and enabling two-factor authentication was critical. There were concerns about potential exposure of sensitive credentials in public repositories, highlighting the importance of using environment variables or .gitignore files to prevent accidental commits of passwords and API keys. This experience reinforced best practices for secure coding and version control in collaborative projects.

For the next assignment, I would like to have more time to format tables and plots properly, as some dimensions still look slightly off compared to the Jupyter Notebook. Improving this will make the visualizations more readable and professional.
